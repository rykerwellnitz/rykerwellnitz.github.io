# Getting rid of unnecessary columns
crimeDF = crimeDF[['Type of Bias','Count of Victims', 'Status of Crime', 'Type of Offense/Crime',
                 'Type of Location', 'Type of Victim', 'Race of Victims', 'Gender of Victim(s)',
                 'Date of Crime', 'Type of Bias.1']]

# Replace NaN values
# I'm not removing rows with missing values for these four as those rows still have 
# valuable information and can still tell us some interesting data.

crimeDF['Race of Victims'].fillna("Unknown", inplace=True)
crimeDF['Gender of Victim(s)'].fillna("Unknown", inplace=True)
crimeDF['Type of Victim'].fillna("Unknown", inplace=True)

# Shortening this bias type, it doesn't read well

# Function to shorten
def shorten_bias(bias_type):
    return 'Other' if bias_type == 'Anti-Lesbian, Gay, Bisexual, or Transgender (Mixed Group)' else bias_type

# Applying the function
crimeDF['Type of Bias'] = crimeDF['Type of Bias'].apply(lambda x: shorten_bias(x))

# Drop nil values for some columns
crimeDF.dropna(subset=['Count of Victims','Type of Bias','Type of Bias.1'], inplace=True)

# Changing column types

# using pd.to_datetime(crimeDF["Date of Crime"]) didn't work
crimeDF['Date of Crime'] = crimeDF['Date of Crime'].apply(pd.to_datetime)
# Removing hours from date
crimeDF['Date of Crime'] = crimeDF['Date of Crime'].dt.date

# Change to integer
crimeDF['Count of Victims'] = crimeDF['Count of Victims'].astype('int')

# Category columns
for col in ['Type of Bias', 'Status of Crime', 'Type of Offense/Crime', 'Type of Location',
           'Type of Victim', 'Race of Victims', 'Gender of Victim(s)']:
    crimeDF[col] = crimeDF[col].astype('category')
    
crimeDF.drop_duplicates(inplace=True)

# I think overall, the data (or at least hate crimes in Tempe) is interesting and there could
# Definitely be some interesting trends. However, this dateset was pretty limited in the amount of
# Entries. This made it difficult to accurately identify trends. It made it VERY difficult to make
# A regression model (and obviously that model was very inaccurate)

# The regression model was interesting. I did have a model with ~0.65 accuracy using 'Type of Bias.1'
# And 'Count of Victims.' The problem is it is difficult to read because Type of Bias.1 is filled with
# Numbers without labels. Also, the scatterplot was inaccurate despite supposedly being around 65% accuracy.

# I do find it interesting that my actual regression model had 0% acurracy, which is why I kept it.
# My heatmap said 'Race of Victims_African American, White' and 'Count of Victims' had around
# A 0.85 accuracy, so a 0.0 score seems impossible. My running theory is something with the dummy
# Variables went wrong along the way and triggered the scorer to say 0. I also struggled with this
# Two modules ago. I feel like I understand it more, but 0.0 is crazy. It could also just be
# Indicating that there really is no correlation between bias and count of victims (Or a lack of data).

# Overall I'm happy with my project despite hiccups with Linear Regression. Next time, I'd probably choose
# A dataset with less categorical variables and more numbers as that caused me some pain.
